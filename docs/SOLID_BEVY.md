# 🛠️ Coding Standards: SOLID for Bevy ECS

## 1. Single Responsibility (S)
*   **System**: Одна система виконує одну логічну задачу (наприклад, `input`, `render`, `cleanup`).
*   **Component**: Один компонент зберігає одну властивість даних.
*   **Files**: Максимальний розмір файлу — **500 рядків**. Якщо файл більший, він має бути розділений на модулі.

## 2. Open/Closed (O)
*   **Plugins**: Будь-який новий функціонал додається через `Plugin`. 
*   **Extension**: Поведінка об'єктів змінюється шляхом додавання компонентів-маркерів, а не через розширення існуючих `enum` або `match` блоків у ядрі.

## 3. Modular Architecture (L/I)
*   **Small Components**: Замість великих структур використовувати набір дрібних компонентів.
*   **Query Filtering**: Системи повинні використовувати точні фільтри (`With<T>`, `Without<U>`), щоб уникнути конфліктів при паралельному виконанні.

## 4. Communication & Dependency (D)
*   **Decoupling**: Системи не викликають одна одну напряму. 
*   **Events**: Використовувати `EventWriter` / `EventReader` для передачі команд між системами.
*   **States**: Переходи між режимами гри здійснюються виключно через `NextState<GameState>`.

## 5. Code Hygiene (Future-proofing)
*   **No Dead Code**: Невикористаний код та імпорти мають видалятися негайно. 
*   **Versioning**: Версія програми має генеруватися автоматично (`YYYYMMDD_N`) через `build.rs`.
*   **Commits**: Заборонено виконувати автоматичні комміти. Будь-яка зміна в репозиторії вимагає явного підтвердження користувача.

---

*Ці правила є обов'язковими для виконання всіма учасниками розробки (включаючи AI).*
