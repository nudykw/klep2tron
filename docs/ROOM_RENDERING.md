# 🏛 Room Rendering Rules / Правила відображення кімнати

Цей документ визначає стандарти візуалізації ігрового простору (кімнат) у Klep2tron.

---

### 1. 📏 Grid and Coordinates / Сітка та координати
- **Розмір**: Кожна кімната має фіксований розмір **16x16** тайлів.
- **Одиниці**: 1 тайл = 1.0 одиниця в просторі Bevy.
- **Висота (`h`)**: Визначає вертикальний рівень тайла. Світова координата Y розраховується як `h * 0.5`.

### 2. 🧱 Geometry Structure / Структура геометрії
Кожен тайл складається з двох частин:

1.  **Top Cap (Верхня частина)**:
    - Висота: **0.5**.
    - Верхня грань завжди збігається з рівнем `h * 0.5`.
    - Використовує основний колір рівня (палітра).
2.  **Foundation (Фундамент)**:
    - Починається від рівня підлоги **Y = -0.5**.
    - Заповнює весь простір до нижньої межі Top Cap.
    - Якщо `h = 0`, фундамент відсутній.
    - Використовує затемнений колір (70% яскравості).

### 3. 📐 Tile Types / Типи тайлів
- **Cube**: Стандартний прямокутний блок.
- **Wedge (N, E, S, W)**: Скошений блок (рампа). 
    - Напрямок вказує сторону **підйому**.
    - Меш розгортається відповідно до напрямку (`WedgeN` = 0°, `WedgeE` = -90° тощо).
- **Empty**: Порожня клітинка. Якщо `h < 0`, тайл не рендериться (дірка в підлозі).

### 4. 🎨 Coloring Logic / Логіка розмальовки

#### 🌈 Rainbow Palette (Веселка)
Колір залежить від висоти `h`:
- H=0: Red, H=1: Orange, H=2: Yellow, H=3: Green, H=4: Blue, H=5: Indigo, H=6: Violet.

#### 🪜 Wedge Inheritance (Успадкування рамп)
- Рампа запозичує колір у сусідньої рампи **того ж типу**, якщо та стоїть вище по схилу.
- Рампа **НЕ** запозичує колір у Кубів.
- Це створює візуально єдині схили одного кольору.

#### 🏁 Chessboard Effect (Шахове затінення)
Для читання глибини до верхньої грані застосовується:
- Парні клітинки `(x + z) % 2 == 0`: Базовий колір.
- Непарні клітинки: Затемнення на 10%.

### 5. 🧹 Cleanup and Transitions / Очищення та переходи
- При виході з кімнати (OnExit `GameState::InGame`) всі об'єкти з компонентом `MapEntity` або `TileEntity` мають бути видалені (`despawn_recursive`).
- Ресурс `TileMap` має бути очищений.

---

### 🛠 Technical Reference
- **Mesh Assets**: `cube_mesh`, `wedge_mesh` (завантажуються як `.obj`).
- **Materials**: Динамічно створюються та кешуються в `map_rendering_system`.
