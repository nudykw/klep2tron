# Формат хранения Актора (Actor Storage Format)

Документ описывает структуру хранения данных персонажа (NPC/Actor) в проекте Klep2tron. Система использует гибридный подход: **Параметрический конфиг** для редактирования и **Бинарные слепки (Bakes)** для производительности в игре.

## 1. Структура папки
Каждый актер хранится в своей подпапке в `assets/actors/[Name]/`:
- `actor.ron` — Основной файл манифеста (Конфиг).
- `head.k2m` — Бинарный меш головы (Bake).
- `body.k2m` — Бинарный меш тела (Bake).
- `legs.k2m` — Бинарный меш ног (Bake).

## 2. Файл манифеста (actor.ron)
Файл в формате RON (Rust Object Notation). Содержит метаданные и инструкции по сборке.

### Основные поля:
- `name`: Имя актора.
- `source_path`: Путь к оригинальной 3D-модели (GLB/OBJ), из которой был создан актер. Позволяет повторно импортировать и перерезать модель в редакторе.
- `cut_top` / `cut_bottom`: Высоты разрезов в нормализованных координатах (0.0 - 1.0).
- `optimization_budget`: Целевое количество треугольников (если применялась оптимизация).
- `scale`: Физический масштаб модели в мире (Vec3).
- `config`: Объект конфигурации, содержащий:
    - `sockets`: Список точек привязки (название, координаты, поворот, эффекты).

## 3. Бинарный формат меша (.k2m)
Собственный высокопроизводительный формат Klep2tron Mesh. Предназначен для мгновенной загрузки в движок Bevy без парсинга и расчетов.

### Спецификация (Little Endian):
1. **Magic Number:** `0x4B324D21` ("K2M!") [4 байта]
2. **Version:** `1` [4 байта]
3. **Vertex Count:** Количество вершин [4 байта]
4. **Index Count:** Количество индексов [4 байта]
5. **Positions:** `[f32; 3]` * Vertex Count
6. **Normals:** `[f32; 3]` * Vertex Count
7. **UVs:** `[f32; 2]` * Vertex Count
8. **Indices:** `u32` * Index Count

## 4. Жизненный цикл данных
1. **Редактор:** Читает `actor.ron`. Если бинарные меши (`.k2m`) существуют, они загружаются мгновенно. Если пользователь меняет параметры (разрез, масштаб), редактор берет оригинал из `source_path`, выполняет расчеты и при сохранении обновляет `.k2m`.
2. **Игра:** Читает `actor.ron`, находит ссылки на `.k2m` файлы, загружает их и расставляет сокеты согласно списку. Математика разрезания в игре не выполняется.
