> **ВНИМАНИЕ (ДЛЯ ИИ):** Этот документ является главным источником истины для разработки проекта "Клептотрон". При получении новых задач ИИ обязан сверяться с этим документом. В случае противоречий между запросом пользователя и требованиями в этом документе, ИИ обязан приостановить работу и задать уточняющий вопрос пользователю.

## 1. Общее описание
- **Название игры:** Клептотрон (Kleptotron).
- **Жанр/Стиль:** 2.5D игра с видом сверху под изометрическим/перспективным углом (похоже на Diablo).
- **Платформы:** Windows, Linux, macOS (нативные приложения). Возможен перенос в Web/Mobile в будущем.
- **Архитектура:** Клиент-серверная. Сервер является авторитарным (считает физику, хранит состояния).
- **Стек технологий:**
  - **Движок:** Rust + Bevy (актуальная версия 0.14).
  - **Физика:** Avian3D.
  - **Управление:** leafwing-input-manager.
  - **Сеть:** bevy_renet (в будущем поддержка WebTransport/WebRTC).
  - **База Данных сервера:** PostgreSQL + SeaORM.
  - **Веб-админка:** Rust + Axum.

## 2. Игровой мир и Механики
- **Уровни и Комнаты:** Игровой мир (Уровень) состоит из комнат. Комнаты соединяются невидимыми порталами.
- **Тайлы:** Комната — это многоуровневая 3D-сетка, состоящая из тайлов.
  - Типы тайлов: `Empty` (пусто), `Flat` (горизонтальный пол), `Wall` (вертикальный блок), `Slope` (наклон под 45 градусов в одну из 4 сторон света).
- **Объекты:** На тайлах могут находиться предметы. Часть из них — препятствия (непроходимые), с другими можно взаимодействовать.
- **Игрок (Робот):** 
  - Форма коллайдера: Цилиндр.
  - Управление: Стрелки на клавиатуре — передвижение. Пробел — прыжок.
  - Физика прыжка: Робот может запрыгнуть на высоту ровно 1 тайл.
  - Геймпады: Управление должно быть абстрагировано (через leafwing), чтобы позже добавить поддержку геймпадов и смену биндов.

## 3. Требования к UI и Локализации
- **Главное меню:** Позволяет "Создать новую игру", "Загрузить игру", перейти в "Настройки".
- **Локализация:** Встроена в клиентскую часть.
  - Поддерживаемые языки: Украинский (по умолчанию), Английский.
  - Язык должен меняться мгновенно (по кнопке L) без перезагрузки интерфейса.

## 4. Требования к Инфраструктуре
- Серверная часть работает без графики (Headless Bevy) на частоте ~60 тиков в секунду.
- Сервер упаковывается в Docker-контейнер и запускается вместе с PostgreSQL через `docker-compose`.
- **Веб-админка:** Отдельное веб-приложение для администраторов. Функционал: создание/редактирование уровней, spectating (наблюдение) за игроками в реальном времени.

## 6. Задачи (Что НУЖНО реализовать)
*Пользователь может менять приоритеты этих задач:*
- [ ] **Добавление Игрока**: Спавн цилиндра в координатах (x, y, z) в комнате при старте.
- [ ] **Физика**: Интеграция `avian3d`. Генерация статических коллайдеров для всех непроходимых тайлов (Flat, Wall, Slope) в комнате.
- [ ] **Управление**: Интеграция `leafwing-input-manager`. Написание Kinematic Character Controller для робота (передвижение стрелками, гравитация, проверка земли лучом для прыжка).
- [ ] **Сетевой слой**: Интеграция `bevy_renet`. Подключение клиента к серверу, передача команд управления на сервер, получение координат от сервера.
- [ ] **База Данных**: Подключение SeaORM к Postgres, создание схемы для сохранения прогресса.
- [ ] **Админка**: Разработка базового API на Axum для просмотра состояния сервера.
