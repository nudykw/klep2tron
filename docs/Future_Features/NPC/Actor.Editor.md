# Character Editor (Редактор Актора)

## 1. Требования

- 1.1. **Визуализация**: Отображение 3D-модели в реальном времени с возможностью вращения (Orbit Camera) и зума.
- 1.2. **Разделение (Splitting)**: Инструментарий для визуального определения границ (Y-координат) разреза модели на части (Голова, Тело, Двигатель).
- 1.3. **Предпросмотр кастомизации**:
    - 1.3.1. Смена мешей для каждой части.
    - 1.3.2. Изменение цвета и свойств материалов (StandardMaterial).
    - 1.3.3. Применение текстур.
- 1.4. **Тестирование анимации**: Кнопки или ползунки для проверки вращения головы независимо от тела.
- 1.5. **Экспорт**:
    - 1.5.1. Сохранение разделенных мешей (GLB).
    - 1.5.2. Сохранение метаданных (конфиг проекта).
- 1.6. **Интерфейс**: Удобные панели управления (UI) на базе `egui`.
- 1.7. **Точки привязки (Sockets)**: Инструментарий для расстановки точек (позиция + вектор) на поверхности модели для спецэффектов (пламя из сопел, точки выстрела, свет глаз).

## 2. Принятые технические решения

- ~~2.1. **Базовый стек и UI**~~ [Вердикт]
    - 2.1.1. **Движок и UI**: Bevy + `bevy_egui` для инспектора и панелей. [Вердикт]
    - 2.1.2. **Камера и навигация**: `bevy_pan_orbit_camera` (Orbit Camera с поддержкой мыши). [Вердикт]
- ~~2.2. **Алгоритм и управление разрезом**~~ [Вердикт]
    - 2.2.1. Использование терминов: **Head**, **Body**, **Engine**.
    - 2.2.2. Единый вертикальный **Range Slider** для задания границ разреза.
    - 2.2.3. **Визуализация**: Полупрозрачные плоскости разреза должны отображаться в 3D-пространстве и вращаться вместе с моделью (привязка к локальным координатам).
    - 2.2.4. **Режим инспекции**: При клике на часть модели, остальные части скрываются, а выбранная часть автоматически масштабируется (Auto-focus) для заполнения окна. В этом режиме доступно свободное вращение и приближение/удаление. Наличие UI-кнопки для возврата к общей модели.
    - ~~2.2.5. **Зашивка разреза (Capping)**~~: Использование алгоритма **Ear Clipping** для триангуляции контура среза. Это обеспечит "цельность" модели при экспорте в GLB. [Вердикт]
- ~~2.3. **Система сокетов**~~ [Вердикт]
    - 2.3.1. **Установка**: Клик по поверхности меша (Picking) ставит сокет, ориентированный по нормали поверхности. Сокет привязывается к локальным координатам части (Head/Body/Engine).
    - 2.3.2. **Манипуляторы**: Использование полнофункционального 3D-Gizmo для перемещения (Position) и поворота (Direction) сокета.
    - 2.3.3. **Интерфейс управления**: Единый контрол в боковой панели с разделением списка на три логические группы (**Head**, **Body**, **Engine**).
    - 2.3.4. **Функционал**: Выбор сокета, прямое редактирование имени и комментария в списке, кнопки добавления и удаления.
- ~~2.4. **Импорт и Экспорт**~~ [Вердикт]
    - 2.4.1. **Импорт**: Поддержка форматов **GLTF**, **GLB** и **OBJ**.
    - 2.4.2. **Экспорт (Game Ready)**: Основной формат — **GLB**. Сокеты и комментарии сохраняются внутри модели (через иерархию Nodes и поле `extras`).
    - 2.4.3. **Проект (Editor State)**: Использование `project.json` для сохранения полного состояния редактора (путь к исходнику, положение ползунков, настройки камеры). [Исследование]
    - 2.4.4. **Структура папок (Output)**: Результат экспорта сохраняется в `assets/3dModels/NPC/{ActorName}/`:
        - `Head.glb` — модель головы и её сокеты.
        - `Body.glb` — модель тела и её сокеты.
        - `Engine.glb` — модель двигателя и её сокеты.
        - `project.json` — файл проекта для повторного редактирования.
    - 2.4.5. **Библиотеки**: Использование крейтов `gltf`, `gltf-json` для работы с 3D-данными и `serde_json` для конфигов.
- ~~2.5. **Доступность и Платформа**~~ [Вердикт]
    - 2.5.1. **Платформа**: Редактор разрабатывается исключительно для десктоп-версий (Linux/Windows). Реализация веб-версии (WASM) не требуется.
    - 2.5.2. **Интеграция**: Редактор должен быть доступен как **отдельный пункт в главном меню** игры.

## 3. Интерфейс (Layout)

- 3.1. **Центральная область**: 3D Вьюпорт для взаимодействия с моделью, отображения Gizmos и плоскостей разреза.
- 3.2. **Левая панель (Управление проектом)**:
    - 3.2.1. Кнопка **"Import Model"**: Выбор исходного файла (GLB/OBJ).
    - 3.2.2. Кнопка **"Open Project"**: Выбор **папки проекта** (автоматическая загрузка `project.json` и связанных ресурсов).
    - 3.2.3. Кнопка **"Save Project"**: Сохранение всего состояния (JSON и ассеты) в выбранную **папку**.
    - 3.2.4. Переключатель режимов: **Slicing** (Разрез) / **Sockets** (Точки).
- 3.3. **Правая панель (Инспектор свойств)**:
    - 3.3.1. **Секция разреза**: Вертикальный Range Slider для управления плоскостями.
    - 3.3.2. **Секция частей**: Список (Head, Body, Engine) с настройками материалов (Color Picker) и кнопкой входа в режим инспекции.
    - 3.3.3. **Секция сокетов**: Группированный список точек с возможностью редактирования имени, комментария и кнопками (+/-).

## 4. План реализации

- 4.1. **Этап 1: Базовый вьюер**
    - 4.1.1. Загрузка исходной модели из `assets/3dModels/NPC/Sources/`.
    - 4.1.2. Реализация Orbit Camera.
- 4.2. **Этап 2: Инструмент разделения**
    - 4.2.1. Визуальные плоскости разреза (Gizmos), которые можно двигать вверх-вниз.
    - 4.2.2. Логика динамического обновления мешей частей при изменении границ разреза.
    - 4.2.3. Реализация алгоритма Ear Clipping для зашивки срезов.
- 4.3. **Этап 3: Настройка материалов**
    - 4.3.1. Color Picker и слайдеры для свойств материалов каждой части.
    - 4.3.2. Интерфейс выбора текстур.
- 4.4. **Этап 4: Система сохранения**
    - 4.4.1. Реализация функций импорта, открытия и сохранения проекта.
    - 4.4.2. Кнопка "Export", которая генерирует три `.glb` файла.
- 4.5. **Этап 5: Система точек привязки (Sockets)**
    - 4.5.1. Реализация расстановки 3D-маркеров на поверхности модели.
    - 4.5.2. Визуальное редактирование векторов направления и позиции для каждой точки.
    - 4.5.3. Сохранение данных о точках внутри GLB (Nodes + Extras).
- 4.6. **Этап 6: Интеграция с игрой**
    - 4.6.1. Возможность загрузить созданный конфиг обратно в игру для тестирования управления и визуальных эффектов.
