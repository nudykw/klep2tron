# 12. Оптимизация моделей (Integrated GPU focus)

> [!IMPORTANT]
> **Контекст проекта:** Игра в стиле 2.5D, где модели NPC состоят из независимых частей (Head, Body, Engine и т.д.), которые могут отделяться (Explodable parts). Костная анимация (Skinning) не используется. Основной метод покраски — Vertex Colors.
> **Цель:** Снижение полигонажа высокодетальных исходных моделей (до 500,000+ полигонов) до игрового бюджета (~15,000 полигонов) для стабильной работы на встроенных GPU (Radeon 680M).

## 1. Функциональные требования (UI & UX)

### Панель "Polycount Budget" (в Инспекторе)
- **Current Triangles:** Отображение текущего количества треугольников (берется из существующего счетчика в статус-баре).
- **Target Triangles:** Поле ввода желаемого количества полигонов (по умолчанию 15,000).
- **Кнопка "Optimize Mesh":** Запуск процесса упрощения всех активных частей модели до указанного лимита.

### Визуальный контроль (Visualization)
- **Wireframe Mode (Toggle):** Кнопка включения сетки поверх модели для оценки искажений топологии.
- **Original/Optimized Toggle:** Возможность быстрого переключения между исходным и упрощенным состоянием для сравнения (A/B тестирование).

## 2. Технические задачи

### Алгоритм упрощения (Decimation)
- [x] Интеграция библиотеки упрощения мешей (`meshopt-rs`).
- [x] **Сохранение Vertex Colors:** Алгоритм корректно интерполирует цвета вершин.
- [x] **Целостность границ:** Оптимизация целой модели перед нарезкой исключает щели на стыках.

### Оптимизация Slicing Caps
- [x] **Douglas-Peucker:** Крышки (Caps) генерируются с минимальным количеством треугольников (упрощение контура).
- [x] **Unified Control:** Реализован единый слайдер **Fill/Rim** (от Solid до Hollow).
- [x] **Результат:** Общая сложность модели снижена в ~3 раза (с 15к до 5к треугольников на срезах).

## 3. Ограничения (Что НЕ делаем)
- **НЕТ LOD-ам:** Так как игра 2.5D и масштаб моделей почти статичен, уровни детализации не требуются. Оптимизация делается один раз.
- **НЕТ Skinning:** Игнорируем любые задачи, связанные с костями и весами вершин.
- **НЕТ Instancing в редакторе:** Фокусируемся только на геометрии одного Actor'а.

## 4. Критерии успеха
- Модель весом 500k треугольников успешно сжимается до 15k за приемлемое время (несколько секунд).
- Визуальные искажения формы при 15k полигонах минимальны для камеры 2.5D.
- Vertex Colors сохраняются без артефактов.
