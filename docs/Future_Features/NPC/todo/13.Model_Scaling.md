# 13. Масштабирование модели (Model Scaling)

> [!IMPORTANT]
> **Контекст проекта:** Для баланса геймплея и корректного отображения в 2.5D мире, все NPC должны соответствовать определенным габаритам (например, высота человека, размер дрона). 
> **Цель:** Позволить пользователю задать точные физические размеры модели в метрах (или игровых единицах) по трем осям.

## 1. Функциональные требования (UI & UX)

### Панель "Model Dimensions" (в Инспекторе)
- **Width (X):** Текстовое поле ввода ширины.
- **Height (Y):** Текстовое поле ввода высоты.
- **Length (Z):** Текстовое поле ввода длины/глубины.
- **Initial State:** При открытии модели поля инициализируются текущими размерами ограничивающего бокса (Bounding Box) модели.
- **Proportional Scaling (Toggle):** Переключатель (замок), при включении которого изменение одного параметра автоматически пересчитывает остальные для сохранения пропорций.
- **Apply Button:** Кнопка применения изменений (или авто-применение при потере фокуса/Enter).

## 2. Технические задачи

### Расчет габаритов
- [x] Реализация системы вычисления общего AABB (Axis-Aligned Bounding Box) для всех видимых частей Actor'а.
- [x] Учет текущего Scale трансформ при вычислении "текущих" размеров.

### Логика масштабирования
- [x] Вычисление коэффициентов масштабирования (Scale Factors) на основе целевых размеров и текущего AABB.
- [x] Поддержка "непропорционального" масштабирования (изменение только одной оси).
- [x] Применение масштаба к корневому объекту или ко всем частям меша.

### Интеграция с другими системами
- [x] **Slicing:** Убедиться, что плоскости среза (Slicing Planes) корректно соотносятся с новым масштабом.
- [x] **Undo/Redo:** Добавление операции масштабирования в историю изменений.
- [x] **Save/Load:** Сохранение примененного масштаба в файл проекта.

## 3. Ограничения
- **Минимальный размер:** Ограничение на минимальный масштаб (например, не менее 0.001), чтобы избежать проблем с плавающей точкой и исчезновением модели.
- **Центрирование:** Масштабирование должно происходить относительно Pivot Point (обычно низ модели, стоящий на земле).

## 4. Критерии успеха
- Пользователь вводит "2.0" в поле Height, и вся модель растягивается/сжимается так, чтобы её верхняя точка была ровно на высоте 2 единицы от пола.
- При включенном "замочке" пропорций, изменение высоты в 2 раза увеличивает ширину и длину тоже в 2 раза.
- Размеры в UI всегда актуальны и показывают реальный физический размер модели в сцене.
