# [x] 13. Масштабирование модели (Model Scaling)

> [!IMPORTANT]
> **Контекст проекта:** Для баланса геймплея и корректного отображения в 2.5D мире, все NPC должны соответствовать определенным габаритам.
> **Результат:** Задача полностью реализована и расширена системой бинарного экспорта для оптимизации загрузки в игре.

## [x] 1. Функциональные требования (UI & UX)

### [x] Панель "Model Dimensions" (в Инспекторе)

- [x] **Width (X), Height (Y), Length (Z):** Текстовые поля ввода физических размеров в метрах (m).
- [x] **Initial State:** Поля инициализируются реальными мировыми размерами модели.
- [x] **Proportional Scaling (Magnet):** Переключатель пропорционального изменения (сохранение Aspect Ratio).
- [x] **Tooltips:** Всплывающие подсказки с указанием единиц измерения.

## [x] 2. Технические задачи

### [x] Расчет и Логика

- [x] Вычисление AABB с учетом нормализации и текущего масштаба.
- [x] **ScaleModelCommand:** Поддержка Undo/Redo для всех изменений масштаба.
- [x] Исправление бага "двойного масштабирования" при загрузке проекта.

### [x] Интеграция и Оптимизация

- [x] **Slicing Fix:** Синхронизация плоскостей среза и контуров при изменении масштаба.
- [x] **Persistent Optimization:** Сохранение бюджета полигонов в `actor.ron`.
- [x] **Bake-to-Disk:** Автоматический экспорт частей тела в бинарный формат `.k2m` при сохранении.
- [x] **Instant Load:** Загрузка пре-компилированных мешей без пересчета математики.

## [x] 3. Итоги

Задача 13 переросла в полноценную систему подготовки модели к игре. Модель теперь не только корректно масштабируется, но и сохраняется в виде готовых к использованию в движке бинарных артефактов.

**Документация по формату хранения:** [Actor_Storage_Format.md](/docs/Actor_Storage_Format.md)
