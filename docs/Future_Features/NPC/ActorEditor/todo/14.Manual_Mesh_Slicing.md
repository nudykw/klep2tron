# Задача 14: Ручное разделение меша (Manual Mesh Slicing)

> [!IMPORTANT]
> Работа ведется исключительно в ветке **feat/actor-editor**.

## Цель

Реализовать систему ручного переназначения треугольников между частями меша (Head, Body, Legs) для исправления ошибок автоматической нарезки плоскостями на сложных моделях (например, чайник со смещенным носиком).

## Требования к реализации

### 1. Переключение режимов (Slicing Methods)

- **Система состояний:** Резка меша теперь имеет два взаимоисключающих состояния:
  - **Plane Mode (Auto):** Текущая логика разделения двумя горизонтальными плоскостями.
  - **Manual Mode (Correction):** Режим статического меша, где треугольники принудительно привязаны к конкретным частям.
- **Логика переходов:**
  - Переход `Auto -> Manual`: "Замораживает" текущее состояние разрезанных мешей и делает их доступными для редактирования.
    - **Опция "Keep Caps" (Оставить крышки):** Чекбокс, доступный при переходе.
      - *Включено (по умолчанию):* Процедурные крышки (заглушки) превращаются в обычную геометрию и прикрепляются к частям меша.
      - *Выключено:* Крышки удаляются, остаются только оригинальные треугольники меша. Это позволяет избежать внутренних "перегородок" при объединении ранее разрезанных частей (например, половинок носика чайника).
      - **Tooltip:** "Сохраняет заглушки на срезе. Отключите, чтобы избежать внутренних перегородок при объединении разрезанных частей."
  - Переход `Manual -> Auto`: Вызывает предупреждение. При подтверждении все ручные правки сбрасываются, и меш снова начинает нарезаться плоскостями.

### 2. Инструмент выделения (Lasso Selection)

- **Экранное Лассо:** Пользователь рисует произвольный контур на экране.
- **Выборка:** Выделяются треугольники, чьи проекции вершин попадают внутрь контура.
- **Приоритет (Hierarchy Filter):** Если лассо одновременно захватывает треугольники разных частей, выделение автоматически фильтруется по самой "высокой" части в иерархии: **Голова > Тело > Ноги**.
  - *Пример:* При захвате области "Голова + Тело" выделятся только треугольники Головы.
- **Глубина (Visibility):** Выделяются только видимые треугольники (Front-face selection). Насквозь модель не прошивается.
- **Сброс выделения:** Клик по пустому месту вьюпорта или клавиша Esc.
- **Режимы:**
  - `Add`: Добавление новых треугольников к текущему выделению. Или просто выделение мышью.
  - `Subtract`: Исключение треугольников из выделения. Или выделение с зажатой кнопкой Alt.
    На кнопках должна висеть подсказка (Tooltip) об альтернативном способе выделения.

### 3. Пользовательский интерфейс (UI)

- **Панель Slicing (Right Sidebar):** Используется для управления нарезкой и выделением.
  - **Блок Auto:** Содержит настройку толщины ободка (Rim Thickness) и текстовые поля для точного ввода значений уровней нарезки (Top/Bottom Cut). Слайдеры во вьюпорте (гизмо) сохраняются для быстрого управления. Слайдеры и текстовые поля должны быть синхронизированы.
  - **Блок Manual:**
    - Инфо: Количество выделенных треугольников. Цвет текста должен быть того же цвета, что и выделенные треугольники во вьюпорте.
    - **Target Part Selector:** Комбобокс (Head / Body / Legs). Выбор значения мгновенно переносит выделенные треугольники в соответствующую часть. Выделение при этом не сбрасывается.
- **Вьюпорт:**
  - Кнопка-переключатель (Toggle) между режимами Auto и Manual.
  - **Скрытие слайсеров:** В ручном режиме визуальные плоскости нарезки полностью скрываются из вьюпорта, чтобы не мешать выделению. Общие гизмо (сокеты, масштабирование) остаются под управлением пользователя (через стандартные переключатели видимости).
  - Чекбокс **"Keep Caps" (Оставить крышки)** рядом с переключателем режима.
  - Принудительное включение отображения сетки (Wireframe) в ручном режиме.
  - **Подсветка выделения:** Реализуется через значительное увеличение яркости (Brightness boost / Glow) выделенных треугольников. Это позволяет видеть, к какой части тела (синий/серый/оранжевый) относится фрагмент, одновременно понимая, что он выбран.

### 4. Ограничения и Безопасность

- **Lock Required:** Ручное редактирование возможно только если Слайсер заблокирован (Locked).
- **Mesh Integrity (Интеграция операций):**
  - Если ручных правок (overrides) еще нет: операции масштабирования и оптимизации работают глобально через оригинальный меш (как в Auto режиме).
  - Если ручные правки применены: операции масштабирования и оптимизации применяются к каждой из 3-х частей (Head, Body, Legs) независимо. Это позволяет сохранять ручную коррекцию (например, носик чайника) даже при изменении полигонажа или размера.
  - При выполнении любой операции из левой панели активное выделение (лассо) сбрасывается.
- **Rim Thickness:** В ручном режиме изменение толщины ободка заблокировано (контрол скрыт или недоступен), так как эта операция требует полной перегенерации геометрии по плоскостям.

### 5. Сохранение и Загрузка

- **Метаданные:** В файл проекта сохраняется флаг последнего использованного метода резки.
- **Загрузка:** Если проект сохранен в Manual режиме, при загрузке слайсеры и плоскости скрываются автоматически.
- **Baking:** Итоговые меши сохраняются на диск стандартным способом как 3 отдельных объекта.

### 6. Undo/Redo

- Действие "Перенос выделения в другую часть" (через комбобокс) должно записываться в стек Undo/Redo.
- Отдельные шаги рисования контура лассо в стек не записываются.
