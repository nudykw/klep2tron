Редактор будет реализован как отдельное состояние Bevy `GameState::ActorEditor`.

### 1.1. Состояния редактора (`enum EditorSubState`)
- `Loading`: Загрузка исходных ассетов.
- `Editing`: Основной цикл правки (разрез, материалы).
- `SocketPlacement`: Режим работы с точками привязки.
- `Exporting`: Процесс генерации мешей и сохранения файлов.

## 2. Сущности и Компоненты (Entities & Components)

### 2.1. Основные сущности
- **ActorModel**: Корневая сущность загруженной модели.
- **ModelPart**: Часть модели (Head, Body, Engine).
    - Компонент `PartTag { Head, Body, Engine }`.
    - Компонент `OriginalMesh`: Хранит ссылку на исходный неразрезанный меш.
- **SocketEntity**: 3D-маркер точки привязки.
    - Компонент `SocketData { name: String, comment: String, part: PartTag }`.
    - Компонент `Pickable`: Для выбора через `bevy_mod_picking`.
- **SlicingPlane**: Визуальный гид разреза.
    - Компонент `PlaneTag { Up, Bottom }`.

### 2.2. Ресурсы (Resources)
- `ProjectResource`: Актуальное состояние проекта.
    - `source_path: PathBuf`.
    - `cut_top: f32`, `cut_bottom: f32`.
    - `selected_socket: Option<Entity>`.
- `EditorUIState`: Состояние интерфейса egui (открытые вкладки, значения слайдеров).

## 3. Технические детали реализации

### 3.1. Алгоритм динамического разреза
- **Vertex Filtering**: В каждом кадре (или при изменении слайдера) система перебирает вершины `OriginalMesh`.
- **Сортировка**: Вершины распределяются по трем новым мешам в зависимости от их Y-координаты относительно `cut_top` и `cut_bottom`.
- **Ear Clipping (Capping)**:
    - Нахождение ребер, пересекаемых плоскостью.
    - Создание замкнутого контура из точек пересечения.
    - Триангуляция контура для создания "заплатки".

### 3.2. Работа с сокетами
- **Placement**: Используем `Raycast` из `bevy_mod_picking` для получения координат клика и нормали меша.
- **Parenting**: Сокет спавнится как Child-сущность соответствующего `ModelPart`. Это гарантирует правильное наследование трансформаций при экспорте.

### 3.3. Экспорт в GLB
- Используем крейт `gltf-json` и `gltf-builder`.
- Каждый `ModelPart` экспортируется как отдельный `Scene` или отдельный файл.
- Сокеты записываются как `Node` без меша, комментарии записываются в поле `extras`.
