### 3.1. Алгоритм динамического разреза

#### Гибридный подход (Preview + Full)

**Preview режим (во время перетаскивания):**
- **Быстрое вычисление контуров**: Только проверка пересечений треугольников с плоскостями
- **Без создания мешей**: Не создаются новые треугольники и крышки
- **Минимальная интерполяция**: Только позиции вершин, без нормалей/UV/цветов
- **Результат**: Сегменты контуров для визуализации (~5-10ms)

**Full режим (при подтверждении):**
- **Vertex Filtering**: Система перебирает вершины `OriginalMesh`
- **Сортировка**: Вершины распределяются по трем новым мешам в зависимости от их Y-координаты относительно `cut_top` и `cut_bottom`
- **Ear Clipping (Capping)**:
    - Нахождение ребер, пересекаемых плоскостью
    - Создание замкнутого контура из точек пересечения
    - Триангуляция контура для создания "заплатки"
- **Результат**: Полные меши частей + контуры (~50-200ms)

#### Механизм подтверждения
- **Перетаскивание внутри круга**: Автоматическое применение при отпускании
- **Перетаскивание за пределами круга**: Требуется клик по кругу для подтверждения
- **Визуальная обратная связь**: Оранжевые контуры (preview) → Красные контуры (final)
```

## Проблема

При перемещении ползунка слайсера происходило медленное обновление:
- Полное разрезание меша (50-200ms) при каждом движении
- FPS падал до 4-14 при перетаскивании
- Плохая визуальная обратная связь для пользователя

## Решение: Гибридный подход

### Архитектура

```
┌─────────────────────────────────────────────────────────────┐
│                    Перетаскивание ползунка                   │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
                    ┌─────────────────┐
                    │ dragging_gizmo? │
                    └─────────────────┘
                         │         │
                    Yes  │         │  No
                         ▼         ▼
              ┌──────────────┐  ┌──────────────┐
              │ Preview Mode │  │  Full Mode   │
              │   (5-10ms)   │  │ (50-200ms)   │
              └──────────────┘  └──────────────┘
                      │                 │
                      ▼                 ▼
              ┌──────────────┐  ┌──────────────┐
              │   Оранжевые  │  │   Красные    │
              │   контуры    │  │   контуры    │
              └──────────────┘  └──────────────┘
```

### Компоненты

#### 1. Preview Mode (Fast Path)
- **Модуль:** `geometry::contour_calculator`
- **Система:** `preview_contours_system`
- **Компонент:** `PreviewContours`
- **Функция:** Быстрое вычисление только контуров без создания мешей
- **Время:** ~5-10ms
- **Цвет:** Оранжевый

#### 2. Full Mode (Complete Path)
- **Модуль:** `geometry::slicer`
- **Система:** `mesh_slicing_system`
- **Компонент:** `SlicingContours`
- **Функция:** Полное разрезание с созданием мешей и крышек
- **Время:** ~50-200ms
- **Цвет:** Красный

### Механизм подтверждения

Использует существующую систему "кружочков":

1. **Перетаскивание начато** → `dragging_gizmo = Some(...)`
2. **Во время перетаскивания** → Preview контуры обновляются
3. **Отпускание внутри круга** → `trigger_slice = true` → Full разрезание
4. **Отпускание вне круга** → `needs_confirm = true` → Ожидание клика

### Система: `preview_contours_system`

**Назначение:** Быстрое вычисление контуров разреза во время перетаскивания ползунка

**Расписание:** `Update`

**Условия выполнения:**
- `dragging_gizmo.is_some()` - идет перетаскивание ползунка
- Модель загружена в редактор

**Входные данные:**
- `SlicingSettings` - текущие значения top_cut/bottom_cut
- `OriginalMeshComponent` - исходный меш модели
- `ActorBounds` - границы модели

**Выходные данные:**
- `PreviewContours` - сегменты контуров для отрисовки

**Производительность:** ~5-10ms для модели с 10K треугольников

**Взаимодействие:**
- Читает: `SlicingSettings`, `OriginalMeshComponent`, `ActorBounds`
- Пишет: `PreviewContours`
- Используется: `draw_slicing_contours_system`
```

### Шаблон для описания нового компонента

```markdown
### Компонент: `PreviewContours`

**Назначение:** Хранение сегментов контуров для preview режима

**Поля:**
- `segments: Vec<[Vec3; 2]>` - массив сегментов контуров
- `is_preview: bool` - флаг preview режима (всегда true)

**Жизненный цикл:**
- **Создание:** При начале перетаскивания ползунка
- **Обновление:** Каждый кадр во время перетаскивания
- **Удаление:** При отпускании мыши или подтверждении

**Использование:**
```rust
commands.entity(entity).insert(PreviewContours {
    segments: calculated_segments,
    is_preview: true,
});
```

**Связанные компоненты:**
- `SlicingContours` - финальные контуры после полного разрезания
```
